对话文本聚类算法

Posted on 2021-10-15 In Nature Language Processing
Symbols count in article: 3.3k Reading time ≈ 3 mins.

1. 背景介绍

聚类算法是常用的无监督学习方法，是能够在没有或者缺少先验知识的情况下挖掘到隐藏数据结构与知识的有效手段。它的概念是：将相似的对象归到同一个簇中，使得同一个簇内的数据对象的相似性尽可能大，同时不在同一个簇中的数据对象的差异性也尽可能地大。对话文本是指如语音&文本对话这种多轮交互式的对话语料，如群聊对话等场景。对话文本聚类就是聚类算法在对话语料方向上的应用，首先将对话文本的自然语言转换成向量等数学信息，形成高维空间点之后再去利用各种距离度量方式计算点与点之间的距离，将距离比较近的文本聚成一个簇。使得相同簇中文本都是表达相近的意图，不同簇中文本表达不同的意图。

对话文本聚类可以帮助我们在庞大语料库中挖掘出有价值的内容，如用户意图话术、客服意图话术以及它们之间的对应关系。这些挖掘出来的文本可以用来扩充相对应的对话机器人，丰富对话机器人的话术与意图使其应答更加流畅、更加智能。也可以用作对话机器人添加新场景时，冷启动寻找新意图与话术的手段。甚至还可以辅助业务同学进行文本标注。

目前主流的文本聚类算法基本单一类型的算法，这类算法对对话文本向量的聚类效果总是不尽人意。具体表现为：

  1. 聚类出的语料质量低，掺杂度高。
  2. 对话样本中出现频次较少的意图容易被忽略，这类小意图文本会散布在多个簇中或者直接被聚类算法视为噪声点，很难单独成簇。这给我们寻找小意图话术带来了巨大困难。

2. 文本单一聚类算法

各种聚类算法由于其实现原理不同，在处理不同分布类型的数据时，会产生不同的聚类结果。下图是sklearn中常用的10种聚类算法在6种不同分布类型的二维空间样本上的聚类结果，图中每一行代表不同分布类型的数据，每一列代表不同聚类算法，聚类结果中不同颜色代表不同的簇。

从上图可以看到单一聚类算法在处理分布类型不同的数据会产生差异化的聚类结果。换句话说，单一的聚类算法并不能完美处理任何类型的数据样本。而word-embedding后的文本向量，是一种高维复杂的数据，并且样本在向量空间中的复杂程度会随着维度增加急剧提升，导致向量空间拥有多种甚至互相参杂的数据分布类型。这是单一聚类算法在高维向量中聚类效果不佳的根本原因。
针对这一问题，我们设计了一种基于两阶段的聚类算法。

何为Wide，何为Deep

Posted on 2021-03-26 Edited on 2021-04-04 In recommendation system
Symbols count in article: 3.7k Reading time ≈ 3 mins.

Wide&Deep是google在2016年提出的推荐模型，在学术界和工业界存在巨大的影响力。可以这么认为，Wide&Deep是传统推荐模型往深度推荐模型发展的桥梁，它的设计理念反映了深度推荐模型发展的主流方向。最近在做商品排序相关的项目，入坑推荐系统，期望能从Wide&Deep模型的结构出发，梳理推荐系统设计的背后动机。

1. 模型结构

Wide&Deep模型，由左侧的Wide部分和右侧的Deep部分组成。Wide部分是广义线性模型，结构非常简单，就是将输入层直接连到输出层，中间没有做任何处理。而Deep部分稍微复杂一些，由Embedding+MLP组成。这个前馈神经网络对一些sparse特征进行低维的dense embedding，然后和dense特征合并，经过多层卷积机计算得到结果。

这是Wide&Deep模型的结构，那么问题来了，google为什么要设计一个这种形式的模型？Wide侧的优点是什么，缺点是什么？Deep侧呢？能不能去掉？能不能用其他结构代替？带着这些问题我们接着往下看。

2. 推荐模型的能力

上一节我们介绍了Wide&Deep的结构，也引出模型为什么要这么设计的问题。这节从推荐模型的本质出发，来聊聊什么是推荐模型，推荐模型需要具备什么能力。

首先，我们希望推荐系统记住什么？能够记住的肯定是那些高频、常见的模式。比如：

  • 50岁中国大妈端午节登录电商网站，网站给她推荐粽子，大概率能成交
  • 25岁美国研究生黑五节登录电商网站，网站给他推荐手机，也大概率会有点击、交易等行为

为什么？因为在训练样本中<50, 女性, 中国, 端午节, 粽子>这类训练样本太多，推荐模型只需记住，下次遇到同样的情况，只需进行相同的推荐行为，就能获得很高的收益。这个就是我们希望推荐系统拥有的“记忆”能力。

NLP中的subword算法及实现

Posted on 2020-01-18 Edited on 2020-03-15 In Nature Language Processing
Symbols count in article: 5.8k Reading time ≈ 5 mins.

最近在做机器翻译相关的工作，发现subword算法在NLP各大任务中无处不在。既然要用到并且用好subword，这里就重点捋一遍关于subword的算法以及几个开源的实现。

1. word、subword和character

在神经机器翻译中，通常有一个固定的词表，并且模型的训练和预测都非常依赖这个词表。在神经网络的训练过程中，需要对词表中每个词做向量表，每个词对应不同的向量，即embedding的过程。每个不同的词对应不同的向量，即使两个词看起来十分相近，但在训练过程词向量没有任何关系。这就导致一个单词因为拥有不同的形态产生不同的词，从而产生大词汇量的问题。

机器翻译的词表是定长的，但是需要实际翻译的词汇是开放的(out of vocabulary)。以前的做法是新词汇添加到词典中，但是过大的词典会带来两个问题：

  • 稀疏问题: 某些词汇出现的频率很低，得不到充分的训练
  • 计算量问题: 词典过大，也就意味着embedding过程的计算量会变大

同时，这种word-level的处理方式并不能通过增大词表真正解决OOV的问题，因为再大的词典不能真正覆盖所有的词汇。

为了处理这个问题，一个思路是将字符当做基本单元，建立character-level模型。character-level模型试图使用26个字母加上一些符号去表示所有的词汇，相比于word-level模型，这种处理方式的粒度变小，其输入长度变长，使得数据更加稀疏并且难以学习长远程的依赖关系。类似的工作可参考Character-Level Neural Machine Translation，实验结论是基于字符的模型能更好处理OOV问题，并且能更好学习多语言之间通用的语素。

word-level模型导致严重的OOV，而character-level模型粒度又太小，那么subword-level的处理方式就应运而生。subword将单词划分为更小的单元，比如”older”划分为”old” 和 “er”，而这些单元往往能应用到别的词汇当中。举个例子：

将词划分成字词的形式，能够大大降低词典的大小。同时，未知词汇能以subword组合的形式表示出来，也能提升词典的表达能力。

迁移学习与增量学习

Posted on 2018-10-06 In deep learning
Symbols count in article: 1.6k Reading time ≈ 1 mins.

最近在看一些增量学习的论文，在学习的过程中不免会遇到增量学习与迁移学习的相关概念的联系和区别，下面是个人的一些总结和理解，只是概念理解层面，没有深入到算法层面，若有不当之处还请留言交流指正。

在传统分类任务中，为了保证训练得到的分类模型具有准确性和高可靠性，由两个基本假设:
1.学习的训练样本和新的测试样本满足独立同分布
2.必须有足够可用的训练样本

1. 迁移学习

迁移学习是指用已存有的知识对不同但相关领域问题进行求解的一种机器学习方法。这里迁移学习放宽了上面的两个基本假设，我们可以迁移已有的知识来解决目标领域中仅有少量有标签样本数据甚至没有学习样本的问题。（当然了，目标领域样本量肯定是越多越好的）。当不同元素共享的因素越多，迁移学习就越容易，就好像你学会了骑自行车，可能就很容易学会骑摩托车，但是学会骑自行车再去骑三轮车，可能就会很不适应。（亲身经历，确实如此，三轮车每次拐弯的时候，可能时因为重心的位置与自行车不一样，总有一种要翻车的感觉，很容易失去平衡）

1.1 迁移学习与domain adaptation(域适应)

这里的域适应是迁移学习中最常见的问题之一，主要是域不同但任务相同，且源域数据有标签，目标域数据没有标签或者只有很少数据有标签。

即域适应主要问题是如何减少源域与目标域不同分布之间的差异。

上图为四个办公室用品分类数据集。均包含了显示器这一类，但是采集方式不同，标签都是相同的。 如何将域A上训练好的模型用在无标签或者少标签的域B上，是domain adaptation 主要研究的问题。

ResNet背后的原理

Posted on 2018-03-24 In deep learning
Symbols count in article: 2.3k Reading time ≈ 2 mins.

最近一段时间在准备校招面试，重新看了ResNet这篇神作以及网上的一些解读，不禁感慨凯明大神真的厉害。在这个过程中，对ResNet产生的背景和动机，以及衍生出来的解决方法有些不同的理解，在这篇文章整理和记录一下。

1. 提出的背景

在传统机器学习模型往深度模型演进的时代，我们通常认为，网络的深度决定了网络的性能。这个观点非常容易理解，深度模型也看做是拥有大量参数的黑盒模型，用来拟合大量的数据。模型深度越深，参数量越多，对数据的拟合能力就越强，那在测试集上表现的性能也就越好。从LeNet->AlexNet->VGG这些网络的发展来看，这个结论似乎是正确的，网络越深，性能也越好。

但随着GPU算力大幅度的提升，网络设计越来越深，但是实验数据发现，更深的网络没能带来更好的性能，反而表现出更高的误差:

从这个实验结果图可以看出，在迭代次数足够多的情况下，56层的网络比20层的网络具有更高的training error和test error。照理deeper is better的理论来说，20层的网络是56层网络的子网络，也就是说56层网络的解空间包含了20层网络，性能应该是≥20层网络的，但事实却相反。不断加深的网络结构，往往会导致各种问题:

  1. 梯度消失、梯度爆炸问题。网络误差的传递是一个链式法则，当层数越深的时候，梯度将以指数形式传播。随着网络层数的加深，梯度消失和梯度爆炸的问题会变得越来越明显
  2. 过拟合问题。随着网络层数的加深，参数变多，神经网络的拟合能力变得很强，这也就意味着其表达出来的函数会更复杂。拟合能力更强的网络在测试集上的表现反而更差
  3. 退化问题。随着网络层数的增加，会出现退化问题。该问题的本质是网络深度的加深会导致更复杂的待学习曲线(严重非凸)，SGD等优化算法对网络权重的优化变得更加困难，进而无法找到最优解。

前两个问题已经存在各种各样的解法，ResNet中的Residual(残差)结构主要是为了解决退化问题，这就是ResNet提出的背景意义

2. 怎么解决退化问题

上一节提到ResNet通过提出残差结构来解决网络变深导致的退化问题，从而通过提升网络的深度来提升整体模型的性能。那这一节主要分析下残差结构为什么能解决退化问题。

卷积核的尺寸问题

Posted on 2017-06-12 In deep learning
Symbols count in article: 445 Reading time ≈ 1 mins.

类似的问题有:

  1. 为什么VGG要用3*3的卷积核？
  2. 为什么ResNet除了最开始用大卷积核降低维度以外，后面的残差结构都用3*3的卷积核?
  3. 3*3的卷积核有什么优势?

这里总结一下，为什么当前的深度网络大多用的是3*3的卷积核？

  1. 3*3是最小的能够捕获像素八邻域信息的尺寸
  2. 两个3*3的堆叠卷基层的有限感受野是5*5；三个3*3的堆叠卷基层的感受野是7*7，故可以通过小尺寸卷积层的堆叠替代大尺寸卷积层，并且感受野大小不变
  3. 多个3*3的卷基层比一个大尺寸filter卷基层有更多的非线性(更多层的非线性函数)，使得激活函数更加具有非线性
  4. 多个3*3的卷积层比一个大尺寸的filter有更少的参数。假设卷基层的输入和输出的特征图大小相同为C，那么三个3*3的卷积层参数个数3x(3*3*C*C)=27C2；一个7x7的卷积层参数为49C2；所以可以把三个3*3的filter看成是一个7*7filter的分解(中间层有非线性的分解, 并且起到隐式正则化的作用)
